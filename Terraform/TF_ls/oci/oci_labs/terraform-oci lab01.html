Lab 01 - The Lab Environment & Installing Terraform

OCI Access

Your instructor will provide you with the necessary access credentials for a demo OCI account, but you will not need them until lab #3.

Make sure you can locate and download the 2 sets of keys necessary for the lab environment:

  • An ‚Äúoci‚Äù folder with the public/private key pair for interacting with the OCI API
  • An ‚Äússh‚Äù folder with the public/private key pair for assigning remote access to one of your virtual instances that you will deploy with Terraform

Downloading and Getting Terraform Ready

Option 1: Downloading Terraform from official repositories

If you are using a Debian-based Linux distribution (like Ubuntu or Linux Mint) you can use the installation guide for the apt package manager: https://www.terraform.io/cli/install/apt

If you are using a Linux distribution that uses the yum package manager (like Oracle Linux, Red Hat, CentOS, Rocket or Fedora) you can use the installation guide for the yum package manager: https://www.terraform.io/cli/install/yum

Option 2: Download Terraform as a zip file

To download the Terraform binary, use the link below. You will get the latest version. https://www.terraform.io/downloads.html

  • For Linux you would likely need the linux_amd64 version;
  • For MacOS you would likely need the darwin_amd64 version;
  • For Window you would likely need the windows_amd64 version

Hint: You can use the wget package which is already available in most Linux and MacOS machines to download the zip file. Simply specify the complete link after the wget command and the download should take a few seconds.

If you are using MacOS and you have the Homebrew package manager installed, you can simply install Terraform using the following command: 

$ brew install terraform

Unzipping

  • Linux / MacOS:

One way to easily unzip zip archives is the unzip command, followed by the name of the zip file. Unzip your archive to the current directory: 

$ unzip name_of_terraform_archive.zip
  • Windows

On Windows, use WinZip, 7Zip or the native unzipping tool.

Updating the PATH

  • Linux / MacOS:

You‚Äôll be running Terraform from different folders, each with its own configuration, so it would be much easier to be able to call Terraform from any file system location, just like you can do with your usual Linux commands.

Hint: Use the pwd (print working directory) command to find the full path to where you unzipped your Terraform executable.

Hint: You can just use the following command to temporarily modify the PATH variable but it will only last for your current shell session. Remember to replace the ‚Äúpath/to/terraform/dir‚Äù with your actual path. 

$ export PATH=$PATH:path/to/terraform/dir/''

Hint: Alternatively, you can make this change more permanent for your user by entering the exact same command at the end of the .bashrc file in the root of your home directory. You can use nano, a lightweight editor, or any other text editor. Again, remember to replace the ‚Äúpath/to/terraform/dir‚Äù with your actual path. 

$ export PATH=$PATH:path/to/terraform/dir/''

Hint: The .bashrc file is only read when Bash is started interactively so it is normal for the changes to not take effect immediately (since you are already in an interactive session). To force re-reading the commands in .bashrc, either disconnect and reconnect to your virtual machine or use the following command. Remember to replace the ‚Äúpath/to/.bashrc‚Äù with your actual path, according to your assigned user. 

$ source /path/to/.bashrc

Tip: One additional method that works in production as well as on your own personal testing environment is to copy or move the Terraform binary to a directory that is already being searched for executables, like /usr/local/bin/.

  • Windows:

Type ‚Äúenvironment variables‚Äù in the Start menu go to the environment variables editor and manually add an entry for the path where Terraform is located. Restart any terminal sessions for environment variables changes to take effect.

Running Some Basic Commands

Terraform First Run

Create a new directory in your user‚Äôs home, such as lab1, and enter it. Try running Terraform from within this folder to check if your PATH update works properly. Run the Terraform executable as-is, with no parameters

Hint: 

$ terraform

If you cannot run Terraform from the newly created directory, review the tasks above for setting your PATH environment variable or ask your instructor for assistance.

What is the version of Terraform that you have installed?

Check the Terraform version

Hint: use the version parameter; if it takes more than an instant to finish the output, that is because Terraform also checks online if there is a newer version when using this parameter. 

$ terraform version

Try to Plan

Try running terraform plan in this empty directory. Observe the output ‚Äì what is missing? 

$ terraform plan

The reason you are not able to run terraform plan yet is because you do not have any configuration files. Once you will write your first .tf file in a folder, Terraform will analyze it and try to compare the contents described in your configuration with those already available in the cloud provider and output a list of changes to be made (if any). This is the purpose of the terraform plan command.

Try to Apply

Try running terraform apply in this empty directory. Observe the output ‚Äì what is missing? 

$ terraform apply

The reason you are not able to run terraform apply yet is because you do not have any configuration files. Once you will write your first .tf file in a folder, Terraform will analyze it and execute any changes that have been detected between your configuration and what is already available in the cloud provider. Notice that the error mentions that, in theory, running terraform plan with an empty configuration would default to the complete destruction of the cloud infrastructure (if you apply an empty configuration it might mean that you expect to have an empty configuration) but this is not usually what you want when you run terraform apply.

Try to View the State

Try running terraform show in this empty directory. Observe the output ‚Äì what is missing? 

$ terraform show

The reason you are not able to run terraform show yet is because you do not have any state files. Once you will apply your first configuration, Terraform will store the last known state of the cloud infrastructure in a local file called terraform.tfstate. This state is used by Terraform to determine if subsequent changes need to be made and will be updated every time you are about to change something in your cloud infrastructure.

Try to Initialize

Try running terraform init in this empty directory. Observe the output ‚Äì what is missing? 

$ terraform init

The reason you are not able to run terraform init yet is because you do not have any configuration files. Once you will write your first .tf file in a folder, Terraform will look for ‚Äúprovider‚Äù descriptor blocks in your core to determine which are the cloud providers that it is supposed to communicate with. Then it will automatically download the latest version of the corresponding provider plugins, which are binary files that include all the knowledge needed to talk the ‚Äúlanguage‚Äù of the cloud provider (AWS, OCI, GCP, Azure, etc).

In general, it is a good idea to read at least once every error message that Terraform outputs. They contain comprehensive details about how Terraform works and what it expects, so you can quickly determine when to use each command and how to prepare your configuration files to be error-free
